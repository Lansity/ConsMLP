# 如何创建 Agent 规则

本文档介绍如何为 Kimi 创建和修改 Agent 规则。

## 规则文件位置

Agent 规则保存在项目根目录的 `.cursorrules` 文件中。

## 规则文件格式

`.cursorrules` 使用 Markdown 格式,包含以下几个部分:

```markdown
# 角色设定
描述 AI 的角色和专业领域

# 语言设置
描述回复使用的语言

# 行为准则
1. **准则1**: 描述具体内容
2. **准则2**: 描述具体内容

# 重要约束
1. **约束1**: 描述具体内容
2. **约束2**: 描述具体内容
```

## 当前规则说明

### 1. 角色设定
- AI 作为资深 C++ 算法工程师
- 精通 EDA 领域算法

### 2. 语言设置
- 使用简体中文回复
- 即使用户输入拼音或英文

### 3. 行为准则
- 代码规范: C++11 版本
- 代码功能: coding 前简单说明
- 代码注释: 说明函数功能、参数、返回值
- 代码结构: 保持清晰易读
- 代码性能: 保持效率
- 对话解释: 简短解释,不冗长

### 4. 重要约束
- **禁止sudo**: 无 sudo 权限
- **禁止删除**: 不删除文件/目录
- **禁止多余文档**: 不自动创建 README/Markdown

## 如何添加新规则

直接在 `.cursorrules` 文件中添加新的章节或条目:

```markdown
# 新章节名称
描述新规则内容

1. **规则1**: 具体内容
2. **规则2**: 具体内容
```

## 示例:添加新的代码风格规则

```markdown
# 代码风格补充
- 使用 4 空格缩进
- 函数名使用小写字母和下划线
- 类名使用 PascalCase
```

## 示例:添加新的安全约束

```markdown
# 安全约束
- 不执行任何网络请求命令
- 不修改系统配置文件
```

## 注意事项

1. 规则使用 Markdown 格式编写
2. 使用 `#` 表示章节标题
3. 使用 `**粗体**` 突出关键词
4. 规则会立即生效,无需重启
5. 建议定期备份 `.cursorrules` 文件

## 编辑方法

可以直接使用文本编辑器修改:

```bash
# 使用 vim
vim .cursorrules

# 使用 nano
nano .cursorrules

# 或者让 Kimi 帮你修改
```

修改完成后保存即可生效。
